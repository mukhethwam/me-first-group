Me First Group Website - cPanel Troubleshooting

Use this guide if your site shows a blank page after uploading to cPanel

Common Issues and Solutions

1. Missing .htaccess File

The .htaccess file is crucial for routing to work correctly with a React app.

  • Make sure you uploaded the .htaccess file from the dist folder
  • Check that the file isn't hidden in cPanel's File Manager (click "Settings" and enable "Show Hidden Files")
  • If missing, create a new .htaccess file in public_html with the content from your local .htaccess file

2. File Permissions Issues

Incorrect file permissions can prevent your site from running.

  • For folders: set permissions to 755 (drwxr-xr-x)
  • For files: set permissions to 644 (-rw-r--r--)
  • In cPanel File Manager, right-click on files/folders and select "Change Permissions"

3. Wrong Document Root

Make sure you're uploading files to the correct directory.

  • The main domain usually points to public_html
  • Subdomains might point to public_html/subdomain
  • Check your domain configuration in cPanel to confirm the document root

4. Check Error Logs

Error logs can provide valuable information about what's going wrong.

  • In cPanel, look for "Error Log" in the Metrics section
  • Check both Apache error logs and PHP error logs

5. Browser Cache Issues

  • Try opening your site in an incognito/private browser window
  • Clear your browser cache completely

6. PHP Version Compatibility

Though this is a static site, cPanel may still process through PHP.

  • In cPanel, go to "PHP Version" and try selecting a different version

Still Having Problems?

If none of these solutions work:

  1. Contact your cPanel hosting provider's support
  2. Provide them with the error logs for more specific assistance
  3. Consider deploying to a static site hosting service like Netlify or Vercel as an alternative
